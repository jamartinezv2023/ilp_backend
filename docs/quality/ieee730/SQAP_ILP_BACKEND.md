# Plan de Aseguramiento de Calidad de Software - ILP Backend

## 1. Propósito
Definir el plan de aseguramiento de calidad para la plataforma ILP Backend.

## 2. Alcance
Aplica a microservicios, librerías comunes, gateway, descubrimiento, pruebas, DevOps, seguridad, arquitectura y despliegue.

## 3. Estándares y Referencias
- IEEE 730
- Pirámide de Cohn
- Patrón AAA
- Screenplay
- DevOps
- IaC
- SonarCloud
- Arquitectura Hexagonal

## 4. Roles
- Ingeniero de calidad
- Desarrollador backend
- Arquitecto de software
- DevOps
- Revisor técnico

## 5. Estrategia de Calidad
- No romper build existente
- Cambios incrementales
- Validación automática
- Evidencia en reportes
- Control de defectos

## 6. Métricas
- Build exitoso
- Cobertura de pruebas
- Bugs abiertos/cerrados
- Duplicación
- Code smells
- Vulnerabilidades
- Complejidad ciclomática
- Cumplimiento de arquitectura

## 7. Gestión de Defectos
Todo defecto debe documentarse con:
ID, módulo, severidad, prioridad, descripción, pasos, resultado esperado, resultado obtenido, evidencia y estado.
