# MVP-21A - Congelacion Arquitectonica

Objetivo:
Plataforma estable para iniciar trabajo de campo y captura de datasets reales.

Se congela:
- Arquitectura hexagonal.
- Spring Boot.
- PostgreSQL.
- Docker.
- CI/CD.
- JaCoCo.
- Checkstyle.
- SpotBugs.
- Quality Gates.
- IEEE 730.
- Piramide de Cohn.
- AAA.
- Screenplay BDD como fase futura.

Queda prohibido:
- Reescribir modulos.
- Cambiar tecnologias.
- Agregar microservicios innecesarios.
- Introducir complejidad accidental.
- Implementar XGBoost, Deep Learning, SHAP o MLOps antes del dataset longitudinal.
