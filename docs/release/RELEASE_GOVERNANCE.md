# ILP Release Governance

## Release Readiness Checklist

- Build successful
- Tests successful
- SonarCloud quality gate passed
- No critical vulnerabilities
- No open Sev0 or Sev1 bugs
- Regression suite executed
- Rollback strategy documented
- Release notes prepared
- Observability validated

## Release Types

- Alpha
- Beta
- Release Candidate
- Production

## Mandatory Evidence

Each release must link:

- Azure Boards work items
- Pull requests
- Pipeline runs
- Test reports
- Known issues
- Rollback notes
